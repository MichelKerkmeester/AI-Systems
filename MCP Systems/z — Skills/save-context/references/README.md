# Save-Context Skill - Workflow Flowchart References

This directory contains reference examples for workflow flowcharts generated by the save-context skill. These references demonstrate the desired output format, visual style, and pattern variations.

## Overview

Workflow flowcharts visualize the conversation workflow phases, showing how a conversation progressed through different stages (research, planning, implementation, verification, etc.). They are automatically generated based on conversation data and integrated into the session context documentation.

## Reference Files

### 1. Linear Pattern (`workflow-linear-pattern.md`)
- **Use Case**: Sequential tasks, bug fixes, feature implementations
- **Phases**: 1-4 distinct phases
- **Structure**: Top-to-bottom flow without branching
- **Box Style**: Single-line boxes (`┌─┐`) only
- **Example**: Investigation → Fix → Verification

### 2. Linear 4-Phase Pattern (`workflow-nested-pattern.md`)
- **Use Case**: Full lifecycle development workflows
- **Phases**: Exactly 4 phases (Research, Planning, Implementation, Verification)
- **Structure**: Sequential top-to-bottom flow
- **Box Style**: Single-line boxes (`┌─┐`) only
- **Example**: Research → Planning → Implementation → Verification

### 3. Parallel Pattern (`workflow-parallel-pattern.md`)
- **Use Case**: Concurrent tasks, multi-file refactoring, distributed work
- **Phases**: > 4 phases with concurrent execution
- **Structure**: Side-by-side boxes with sync points
- **Box Style**: Narrow 10-char boxes for parallel tasks
- **Special**: Section dividers and sync annotations

## Design Standards

### Box Formatting
- **Standard width**: 56 characters
- **Parallel width**: 10 characters (for side-by-side layout)
- **Padding**: 2 spaces from box edges
- **Content**: 2-4 activity bullets per phase

### Box Types

#### Terminal (Start/End)
```
╭────────────────────────────────────────────────────────╮
│                  CENTERED TITLE                          │
╰────────────────────────────────────────────────────────╯
```

#### Process Phase (All workflow phases)
```
┌────────────────────────────────────────────────────────┐
│  Phase Name                                            │
│  • Activity 1                                          │
│  • Activity 2                                          │
│  Duration: X minutes                                   │
└────────────────────────────────────────────────────────┘
```

### Parallel Execution Block
```
──────────────────────────────────────────────────────────
SECTION LABEL - Context info
──────────────────────────────────────────────────────────
                        │
      ┌─────────────────┼─────────────────┐
      │                 │                 │
      ▼                 ▼                 ▼
┌──────────┐      ┌──────────┐      ┌──────────┐
│ Phase 1  │      │ Phase 2  │      │ Phase 3  │
│ • Task   │      │ • Task   │      │ • Task   │
└──────────┘      └──────────┘      └──────────┘
      │                 │                 │
      └─────────────────┼─────────────────┘
                        │
                        ▼    (All complete)
```

## Pattern Detection

The save-context skill automatically detects the appropriate pattern based on phase count:

1. **Linear**: ≤ 4 phases — Sequential top-to-bottom progression
2. **Parallel**: > 4 phases — Concurrent execution with synchronization points

All workflows use single-line boxes (`┌─┐`) for consistency and data-driven rendering.

## Integration

These flowcharts are integrated into the context documentation at:
- **Location**: Section 2 (if workflow detected)
- **Template**: `templates/context.template.md`
- **Generator**: `scripts/generate-context.js` (`generateWorkflowFlowchart()`)
